# Foobow PA Audit - 2026-06-07

## Scope

- Target: Foobow static prototype.
- Environment: Playwright Chromium-rendered prototype screenshots and automated interaction suite.
- Attempted live server/browser path: local HTTP startup was blocked by the current shell/browser environment. The in-app browser rejected direct `file:` navigation, and the approval layer rejected an unsafe browser-process server helper. The audit therefore used the committed Playwright visual baselines and behavior suite, which still render the product in Chromium across mobile and desktop.

## Findings

### P1 - Mobile bottom navigation overlaps primary content

- Evidence: Mobile Today screenshot shows the bottom navigation covering the `Complete deed` button.
- Impact: Core activation flow is visually blocked on the main mobile viewport.
- Recommended fix: Increase bottom safe-area padding for `.app-shell`, reserve fixed nav height, and add per-screen lower spacing so primary CTAs cannot sit under `.bottom-nav`.

### P1 - Community title clips on mobile and desktop

- Evidence: Mobile and desktop dark Community screenshots show `A low-pressure kindness wall.` clipped at the left edge.
- Impact: The app fails the first-viewport polish standard and looks broken in a core social screen.
- Recommended fix: Review desktop grid margins and mobile overflow behavior for `.section-heading`, `h2`, and `#screen-community`; ensure headings have stable width, no negative/edge overflow, and responsive line wrapping.

### P2 - Mobile Community list is partially hidden by bottom navigation

- Evidence: Mobile dark Community screenshot shows the third blessing card disappearing behind the bottom nav.
- Impact: Social feed content and actions can be obscured.
- Recommended fix: Apply the same bottom-content reservation as the Today fix and add visual regression coverage for the lower part of the Community screen.

### P2 - Mobile Map filter area feels cramped

- Evidence: Mobile Map/Nature screenshot shows filters packed near the lower viewport and the bottom nav peeking into the frame.
- Impact: The map experience is usable but visually less premium than the target standard.
- Recommended fix: Give map detail/filter content more breathing room and reserve bottom nav spacing.

### P2 - Desktop Today has excessive empty vertical space

- Evidence: Desktop Today screenshot has a large blank gap between content cards and the bottom nav.
- Impact: Desktop layout feels sparse and less enterprise-polished.
- Recommended fix: Consider a desktop-only layout that anchors secondary panels or expands useful content instead of leaving unused space.

## Passing Areas

- Visual identity is calm, premium, and distinctive.
- Donation copy avoids buying luck, virtue, or guaranteed karma.
- Core behavior suite covers check-in, map, deeds, community, profile, donation prompt, language, theme, persistence, keyboard traversal, and contrast.
- Desktop Today card alignment is clean.
- Visual regression baselines now make layout regressions reviewable.
